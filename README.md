*Purpose
The dashboard tracks and analyzes IT helpdesk tickets — helping identify workload, ticket resolution efficiency, user satisfaction, and key performance trends.

*Key Sections Explained
1)Header (Top Section)
2)Title: “IT Ticket Analysis” – communicates focus clearly.
3)KPIs:
-Tickets (9542) → total tickets created.
-Open (6838) → currently unresolved tickets.
-Closed (233) → completed tickets.
-Waiting (1188) → pending or on-hold tickets.
-Avg. Open Days (19) → average time a ticket stays open.
-Tabs (Tickets | Open Days | Avg. Open Days): likely allows toggling between different views.
-Last Refresh Date: ensures users know data is up to date.

*Filters (Left Panel)
1)Multiple dropdown filters for owner_group and Ticket Ageing.
2)Let users narrow the analysis by team, group, or ticket duration.

*Main Visuals
1)Select Measures by Month (Area Chart)
-Shows ticket trends from March to October.
Helps identify spikes or dips in volume over time.
2)Select Measures by Issue Category (Bar Chart)
-Categories like Access/Login, Systems, Hardware, Software.
-Useful for knowing which issue types are most common.
3)Select Measures by Satisfaction Status (Bar Chart)
-Displays user satisfaction levels (Satisfied, Highly Satisfied, etc.).
-Tracks customer feedback quality.
4)Select Measures by Severity (Donut Chart)
-Breakdown of ticket criticality (High, Medium, Low, Unassigned).
-Shows how urgent or severe issues are distributed.
5)Select Measures by Ticket Type (Donut Chart)
-Request vs. Issue — helps distinguish service requests from problem reports.
6)Select Measures by Owner Group (Donut Chart)
-Distribution of tickets handled by groups (Software, Hardware, Networking, Security, Architecture).
-Reveals workload distribution.

*Design & UI
1)Color Theme: Navy blue background with yellow and orange accents — provides strong contrast and aligns with an IT/helpdesk theme.
2)Consistency: Each visual uses similar design structure and layout.
3)Readability: Clear labels, grouped filters, and well-spaced visuals make it easy to interpret.

*Insights You Can Derive
1)Identify which teams or issue types handle most tickets.
2)Measure ticket resolution performance (open vs. closed vs. waiting).
3)Track seasonal trends in ticket creation.
4)Evaluate user satisfaction to improve service quality.
5)Monitor critical issue distribution by severity.
